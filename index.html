📚 Návod: Nastavenie Coolify na Raspberry Pi

🔧 Základné informácie

Coolify je nástroj na samospravované nasadenie aplikácií. Môže bežať priamo na RPi (Localhost) alebo spravovať vzdialené servery (Remote). SSH kľúče sú kľúčové pre bezpečný prístup bez hesla.

🔐 1. Generovanie SSH kľúča (na PC)

Spusti v termináli:

ssh-keygen -t ed25519 -C "tvoje_meno@domena.sk"
  • Uloženie: `~/.ssh/id_ed25519` (stlač Enter)
  • Passphrase: môžeš nechať prázdnu

Vytvorí sa:

  • id_ed25519 → súkromný kľúč (na PC)
  • id_ed25519.pub → verejný kľúč (na RPi)

📤 2. Prenos verejného kľúča na RPi

Spôsob A: Automaticky (odporúčané)

ssh-copy-id -i ~/.ssh/id_ed25519.pub pi@192.168.100.8

Zadaj heslo používateľa pi.

Spôsob B: Manuálne

Na RPi:

mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys

Vlož obsah súboru id_ed25519.pub z PC. Ulož: Ctrl+O → Enter → Ctrl+X.

🔑 3. Nastavenie oprávnení na RPi

chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

🧪 4. Test pripojenia

Na PC:

ssh -i ~/.ssh/id_ed25519 pi@192.168.100.8

Ak si prihlásil bez hesla → všetko funguje!

Poznámka: Ak chceš použiť root, uprav na RPi:

sudo nano /etc/ssh/sshd_config

Nastav:

PermitRootLogin yes

Potom reštartuj SSH:

sudo systemctl restart ssh

🖥️ 5. Nastavenie Remote Server v Coolify

PoleHodnota
Host192.168.100.8

Private KeyObsah ~/.ssh/id_ed25519 (z PC)

💡 Coolify nepoužíva coolify.pem, ak si ho nevytvoril – použi svoj vlastný kľúč.

🔄 8. Reset Coolify (čistá inštalácia)

Na RPi:

cd ~/.coolify
docker compose down
docker volume rm coolify_db coolify_appdata
docker compose up -d

Otvor v prehliadači: http://192.168.100.8:8000 → nový setup

🧩 9. Dôležité poznámky

  • Localhost: žiadny SSH kľúč sa nevytvorí – Coolify beží priamo na RPi.
  • Remote Server: Coolify môže vygenerovať kľúč, ale odporúča sa použiť vlastný.
  • Súbor coolify.pem je vo vnútri Docker kontajnera – nie na tvojom PC.
  • Na PC použi svoj id_ed25519 ako private key v Coolify.

🛑 10. Riešenie problémov

❌ Nejde SSH?

sudo systemctl status ssh
sudo ss -tulnp | grep :22

❌ Nejde Coolify?

curl http://localhost:8000
sudo ss -tulnp | grep 8000

❌ Zablokovaný v setup-e?

Použi reset z bodu 8.
